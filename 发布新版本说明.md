# 发布新版本指南

本文档说明如何构建、发布新版本，使已安装的用户能自动检测到更新。

---

## 一、更新版本号

打开 `package.json`，修改 `version` 字段：

```json
{
  "version": "1.3.0"
}
```

版本号遵循语义化版本规范 `主版本.次版本.修订号`，新版号必须 **高于** 当前版本。

---

## 二、构建安装包

### 方式 A：直接构建（推荐）

```bash
npm run build
```

产物在 `dist/` 目录下：
- `AI Chat Hub Setup x.x.x.exe` — 安装程序
- `latest.yml` — 更新检测文件（自动生成）
- `win-unpacked/` — 免安装版（可选）

### 方式 B：构建并自动发布到 GitHub（需配置 Token）

```bash
# Windows PowerShell
$env:GH_TOKEN="你的GitHub个人访问Token"

npm run build:release
```

> 需要 `GH_TOKEN` 环境变量，Token 需要 `repo` 权限。

---

## 三、手动发布到 GitHub Releases

如果你的构建产物已在本地，按以下步骤手动创建 Release：

### 1. 进入 GitHub 仓库页面

打开 `https://github.com/Shawnylin/aichathub`

### 2. 创建新 Release

- 点击右侧 **Releases** → **Draft a new release**
- **Tag:** 输入 `v1.3.0`（与 `package.json` 版本号一致，前面加 `v`）
- **Release title:** `v1.3.0`
- **Description:** 填写更新日志（示例见下文）

### 3. 上传文件

上传以下文件：
1. `dist/AI Chat Hub Setup x.x.x.exe`
2. `dist/latest.yml`

> ⚠️ `latest.yml` **必须**上传，`electron-updater` 依赖它检测版本。

### 4. 发布

点击 **Publish release**。

## 五、验证更新是否生效

### 方法 1：手动检查

1. 构建并安装旧版本（或保持当前已安装的版本）
2. 发布新 Release 后，打开已安装的客户端
3. 进入 **设置 → 关于**，点击 **检查更新**
4. 应弹出顶部通知栏显示「发现新版本 vx.x.x」

### 方法 2：自动检测

已安装的客户端会在启动 5 秒后自动检测更新（仅限打包后的 .exe 版本，开发模式不会自动检测）。

---

## 六、常见问题

**Q: 检查更新提示「已是最新版本」但明明发布了新版本？**

A: 检查以下原因：
- 确保上传了 `latest.yml` 文件到 Release
- 确保本地安装的版本号低于 Release 版本号
- Release 发布后可能需要等待几分钟 GitHub API 缓存刷新

**Q: 构建时出现 403 权限错误？**

A: 检查 `GH_TOKEN` 是否正确设置，Token 需有 `repo` 权限。

**Q: 开发模式下能测试更新吗？**

A: 开发模式（`npm start`）下 `autoUpdater` 不会生效。如需测试，需先打包安装。

## 七、完整流程速查

```bash
# 1. 修改 package.json 版本号

# 2. 构建安装包
npm run build

# 3. 前往 GitHub Releases 页面创建新 Release
#    - Tag: v新版本号
#    - 上传: AI Chat Hub Setup x.x.x.exe + latest.yml
#    - 填写更新日志 → 发布

# 4. 用户打开客户端即自动收到更新提示
```
